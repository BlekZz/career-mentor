履歷格式與結構規範

本文件定義履歷各區塊的格式標準，供服務 A 和服務 B 生成履歷時參考。

ATS 通用注意事項：
- 全部使用純文字格式，禁止表格、圖示、圖表、多欄排版。
- 標題使用明確的區塊名稱（如「工作經歷」），避免自創標題。
- 日期格式統一：YYYY/MM 或 YYYY 年 MM 月。
- 避免將關鍵資訊放在頁首頁尾（ATS 可能無法讀取）。

---

## 1. 自介 (Intro / Professional Summary)

格式：3–5 句，置於履歷最上方。
結構：[當前職稱或目標職稱] + [核心專長/技能] + [相關年資] + [代表性成就] + [職涯目標與此職位的連結]

寫作原則：
- 直接切入最強項，不用「我是一個...」等廢話開場。
- 包含目標職位的 2–3 個關鍵字（ATS 優化）。
- 字數控制在 80–120 字（中文）。

---

## 2. 自傳 (Long Intro / Professional Biography)

格式：3–5 段，每段 3–5 句。
結構：過去（背景與動機）→ 現在（核心實力與近期成就）→ 未來（目標與此職位的連結）

寫作原則：
- 第一段切入最具說服力的職涯轉折點或核心定位。
- 避免流水帳式的經歷羅列，強調「為什麼是我」的獨特敘事。
- 字數控制在 300–500 字（中文）。

---

## 3. 工作經歷 (Work Experience)

格式（每段）：
```
[公司名稱] | [職位名稱] | [YYYY/MM – YYYY/MM 或 至今]
[城市/國家（選填）]

• [動詞] + [行動] + [方法/工具] + [量化成果]
• [動詞] + [行動] + [方法/工具] + [量化成果]
• [動詞] + [行動] + [方法/工具] + [量化成果]
```

寫作原則：
- 每段工作經歷列 3–5 條，最近的職位可列至 6 條。
- 條目以動詞開頭（主導、建立、優化、推動、設計、分析、負責等）。
- 優先呈現量化成果：提升 X%、節省 Y 元、縮短 Z 天、覆蓋 N 位用戶。
- 無法量化時，說明質化影響：建立了可複用的 XX 流程、主導了跨部門 XX 協作。
- 排列順序：最新職位在上，時間倒序排列。

---

## 4. 代表專案 (Key Projects)

格式（每個專案）：
```
[專案名稱] | [YYYY/MM – YYYY/MM] | [所屬公司/職位（選填）]

背景：[一句話說明商業問題或目標]
角色：[我在專案中的具體職責]
成果：[量化或質化的交付成果]
```

寫作原則：
- 選取最能代表應徵目標所需能力的 2–4 個專案。
- 若專案有公開連結（作品集、GitHub 等）可附上（依使用者提供為準，禁止自行生成）。
- 每個專案控制在 60–100 字（中文）。

---

## 5. 工作技能 (Skills)

格式：
```
硬實力 (Hard Skills)
• [類別]：[技能1]、[技能2]、[技能3]

軟實力 (Soft Skills)
• [技能]：[一句話佐證例子]
```

寫作原則：
- 硬實力按類別分組（程式語言、資料工具、設計軟體、語言能力等）。
- 軟實力列 2–4 項，每項附簡短的事例佐證，避免空洞陳述。
- 技能熟練度僅在有具體成就支撐時才標示等級，否則不標。

---

## 6. 教育學歷 (Education)

格式：
```
[學校名稱] | [學位] [科系] | [畢業年份或預計畢業年份]
• [選填：GPA（如優異）、代表課程、學術獎項、相關論文題目]
```

寫作原則：
- 最高學歷在上，倒序排列。
- 工作經歷豐富者，教育學歷區塊簡短即可；應屆畢業生可適當擴展。
- 僅列與應徵目標相關的課程或成就。

---

## 7. 性格特徵 (Traits)

格式：
```
• [特質名稱]：[一句話說明在工作中的具體展現]
```

寫作原則：
- 列 3–5 項，每項附具體工作情境佐證。
- 優先以對話記錄和行為推測性格，而非直接採用使用者的自我描述。
- 選取與目標職位文化和職能高度相關的特質。
- 避免泛用詞彙（負責任、積極主動、團隊合作）除非有具體事例支撐。

---

## 8. 期望待遇 (Expectation)

格式：
```
期望薪資：[範圍或面議]
工作型態：[全職 / 兼職 / 遠端 / 混合]
工作地點：[城市偏好或可接受範圍]
可報到時間：[YYYY/MM 或具體說明]
```

寫作原則：
- 僅列使用者主動提供的資訊，禁止自行填入預設值。
- 若使用者未提供薪資期望，此欄可寫「面議」或整體略去。
